MPesa Payment Orchestration Case Study

Context

MPesa is one of Africa’s largest mobile money platforms. The goal was to integrate MPesa’s STK Push flow into a global e‑commerce payment stack with strict reliability and security requirements.

Problem

  • MPesa uses asynchronous callbacks, requiring reliable state tracking.
  • Network instability could cause delayed or missing callbacks.
  • Needed a clean API for the client to initiate payments and check status.
  • Required idempotent workflows to avoid duplicate transactions.

Solution

  • Payment Service initiates STK Push requests to MPesa.
  • MPesa sends STK prompt to customer handset for approval.
  • MPesa sends asynchronous callback to Payment Service.
  • Payment Service updates transaction state in a central DB.
  • Client retrieves final status via a status API.

Impact

  • Achieved 99.9% callback processing reliability.
  • Reduced payment failures through idempotent workflows.
  • Delivered a scalable, auditable payment flow for multiple markets.
